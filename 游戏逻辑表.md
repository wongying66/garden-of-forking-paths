# 《寻找X》游戏核心逻辑表

## 一、整体架构：3-1-N-1 结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              3 - 1 - N - 1 结构                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   3  序章：花园入口                                                          │
│      └─> 与3个角色对话，收集3条线索（name/time/location）                      │
│                                                                             │
│   1  无限图书馆 → 发现阿莱夫                                                  │
│      └─> 与僧侣/刺客/将军对话，任一角色引导至阿莱夫                            │
│                                                                             │
│   N  阿莱夫枢纽：随机3个未完成世界 → 完成获得线索 → 返回阿莱夫循环              │
│      └─> 12个世界，每次随机展示3个，完成后移除随机池（可重入，带✦标记）          │
│                                                                             │
│   1  终章：回到花园，与X会合                                                  │
│      └─> 碎片≥10触发木门 → 回到花园 → X对话 → 3种结局                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、线索系统

### 3.1 线索类型

| 类型 | ID格式 | 数量 | 获取方式 |
|------|--------|------|----------|
| 序章线索 | `name`, `time`, `location` | 3条 | 与序章3角色对话获得 |
| 世界线索 | `blade_runner`, `cthulhu`, ... | 12条 | 完成世界自动获得 |
| 隐藏线索 | `*_hidden` | 12条 | 世界内特定选择触发 |

### 3.2 有效碎片计算 (countValidFragments)

```javascript
有效碎片 = clues中符合以下条件的条目：
  - 属于12个世界ID之一（blade_runner, cthulhu, alice, middle_earth, 
    wuxia, cowboy_bebop, matrix, got, spirited, wake, watchmen, maus）
  - 或以 "_hidden" 结尾的隐藏线索

// 只计算这24种有效碎片，序章的3条线索不计入碎片数
```

### 3.3 碎片与进度对应

| 碎片数 | 阶段 | 任务提示 |
|--------|------|----------|
| 0-2 | 序章 | 收集线索（与花园3人交谈） |
| 3-9 | 阿莱夫探索 | 穿越世界，收集碎片（X/10） |
| ≥10 | 终章解锁 | 揭示真相，走向X所在之处 |

## 四、序章流程逻辑

```
开始
  │
  ▼
┌─────────────────┐
│   start（入口）  │
│  旁白叙述 + 4选择 │
└─────────────────┘
  │
  ├─> 走向僧人 ──> meet_monk ──> 多条分支 ──> 获得 clue: name
  │
  ├─> 走向女子 ──> meet_beauty ──> 多条分支 ──> 获得 clue: time
  │
  ├─> 走向猛虎 ──> meet_tiger ──> 多条分支 ──> 获得 clue: location
  │
  └─> 原地回忆 ──> memory_attempt ──> 引导至3角色之一

// 关键：每条分支最终都导向 crossroads（花园中心）
// 无论选择哪条路径，都能获得至少1条序章线索
```

### 4.1 序章线索获取保障

| 角色 | 关键节点 | 线索 |
|------|----------|------|
| 僧人 | monk_inquiry / monk_photos / monk_time / monk_who / monk_where / monk_exist | `name` |
| 女子 | beauty_remember / beauty_full_memory / beauty_meet_truly / beauty_async / beauty_restore / beauty_other_way | `time` |
| 猛虎 | tiger_fast / tiger_sure / tiger_death / path_death | `location` |

**设计原则**：每个角色的所有主要分支最终都包含线索获取，确保玩家不会卡关。

### 6.2 阿莱夫选择逻辑 (aleph_choice)

```javascript
// 每次进入 aleph_choice 时：

1. 获取未完成世界列表 = ALEPH_WORLDS - completedWorlds
2. 从未完成世界中随机选3个展示
3. 如果未完成世界 > 3，提供「换一批」选项
4. 如果 completedWorlds.length > 0，提供「重新进入已探索世界」选项
5. 如果 countValidFragments() >= 10，提供「走向X所在之处」选项
6. 始终提供「离开阿莱夫，回到花园」选项
```

### 6.3 世界完成逻辑 (completeWorld)

```javascript
function completeWorld(worldId) {
    if (!completedWorlds.includes(worldId)) {
        completedWorlds.push(worldId);        // 标记完成
        const world = ALEPH_WORLDS.find(w => w.id === worldId);
        if (world.clue && !clues.includes(world.clue)) {
            clues.push(world.clue);            // 自动给予世界线索
        }
    }
}
```

### 6.4 隐藏线索逻辑 (grantHiddenClue)

```javascript
function grantHiddenClue(worldId) {
    const world = ALEPH_WORLDS.find(w => w.id === worldId);
    if (world.hiddenClue && !clues.includes(world.hiddenClue)) {
        clues.push(world.hiddenClue);          // 给予隐藏线索
        return true;
    }
    return false;
}
// 触发条件：世界内特定对话选择（由世界章节脚本定义）
```

### 6.5 已完成世界重入 (aleph_revisit)

```javascript
// 展示所有已完成世界，带 ✦ 已探索标记
// 玩家可重新进入，但：
//   - 不会重复获得已得线索
//   - 可尝试获取之前遗漏的隐藏线索
//   - 世界剧情可再次体验
```

## 七、终章逻辑

### 7.1 终章触发条件

```
countValidFragments() >= 10
  │
  ▼
aleph_choice / aleph_return 中出现选项：
「★ 碎片已足够——走向X所在之处」
  │
  ▼
final_chapter（终章入口）
```

### 7.2 终章流程

```
final_chapter（阿莱夫中的木门）
  │
  ▼
final_return_garden（回到花园 - 但花园已变）
  │
  ▼
final_meet_x（与X会面 - 核心场景）
  │
  ├─> 有隐藏线索 ──> final_who_is_x_deep（深层版本）
  │                     │
  │                     ├─> 接受 ──> ending_accept_deep
  │                     ├─> 拒绝 ──> ending_reject_deep（环形废墟）
  │                     └─> 融合 ──> ending_merge_deep
  │
  └─> 无隐藏线索 ──> final_who_is_x_shallow（表层版本）
                        │
                        ├─> 接受 ──> ending_accept
                        ├─> 拒绝 ──> ending_reject（环形废墟）
                        └─> 融合 ──> ending_merge
```

### 7.3 隐藏线索对终章的影响

| 维度 | 无隐藏线索（表层） | 有隐藏线索（深层） |
|------|-------------------|-------------------|
| X的态度 | 神秘、距离感 | 温柔、理解、认可 |
| X的台词 | "你终于来了。" | "我在等一个'真正看到我的人'。" |
| 真相揭示 | 标准版：X是你自己 | 深度版：X是敢于看到深处的自己 |
| 结局文本 | 普通结局 | 深度结局（标题带"深处"） |
| 环形废墟 | 标准拒绝结局 | 深层拒绝结局（看到门上铭文） |

## 十、关键设计原则

### 10.1 "固定限制里做文章"

| 场景 | 限制 | 变化 |
|------|------|------|
| 序章 | 3个角色 | 每个角色多条分支，不同特质 |
| 图书馆 | 3个角色 | 每个角色引导至阿莱夫的不同路径 |
| 阿莱夫 | 随机3个世界 | 每次不同组合，已完成可重入 |
| 终章 | 3种选择 | 隐藏线索有无 → 6种变体 |

### 10.2 环形废墟结构

```
起点：花园入口 ──> ... ──> 终点：花园中心
                    │
                    └─> 拒绝结局 ──> 推开门 ──> 回到起点
                    
// 终章的花园 = 序章的花园，但已留下世界印记
// 木门 = 不透露谜底的入口
// "终点即起点" = 博尔赫斯式循环
```

### 10.3 隐藏线索的深层影响链

```
世界内特定选择 ──> grantHiddenClue() ──> clues.push('*_hidden')
                                              │
                                              ▼
终章 getDiscoveredHiddenTruths() ──> 影响X对话内容
                                              │
                                              ▼
final_meet_x 分支 ──> hasHidden ? 深层版本 : 表层版本
                                              │
                                              ▼
结局选择 ──> ending_*_deep（有隐藏）/ ending_*（无隐藏）
```
