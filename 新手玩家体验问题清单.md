## P0 — 游戏无法进行

### P0-1：`vn.html` 缺少 `scenes.js` 和 `main.js` 的引用

- **严重程度**：P0（潜在阻断）
- **代码位置**：`vn.html` 第 911-930 行（script 标签列表）
- **问题描述**：`vn.html` 仅引用了 `vn-engine.js` 和各章节文件，但 **未引用 `js/scenes.js` 和 `js/main.js`**。经过分析，`scenes.js` 使用 `SCENES` 格式（`bg` 属性），`main.js` 依赖 `SCENES` 和 `GameEngine` 来渲染场景。当前 `vn.html` 实际使用的是 `vn-engine.js` 中的 `SCRIPT` 格式和 `VNEngine` 对象，因此 `scenes.js` 和 `main.js` 是**遗留代码**，不影响当前游戏运行。但它们的存在会造成混淆。
- **修复建议**：如果确认不再使用 `scenes.js` / `main.js`，应将其移动到 `_legacy/` 目录或添加注释标记为废弃。避免未来开发者误以为这些文件是必需的。

### P0-2：`scenes.js` 中引用的场景 key 与 `ASSETS.scenes` 不匹配

- **严重程度**：P0（如果游戏使用 `scenes.js` 则会阻断）
- **代码位置**：
  - `js/scenes.js` 第 10、73、97、196 等行：`bg: 'doorBooks'`、`bg: 'doorDesert'`、`bg: 'doorOcean'`
  - `js/vn-engine.js` 第 9-79 行：`ASSETS.scenes` 中不存在 `doorBooks`、`doorDesert`、`doorOcean` 键
- **问题描述**：`scenes.js` 使用了 `doorBooks`、`doorDesert`、`doorOcean` 三个场景 key，但 `ASSETS.scenes` 中没有定义这些 key（虽然根目录有 `bg-door-books.jpg`、`bg-door-desert.jpg`、`bg-door-ocean.jpg`）。当前游戏使用 `vn-engine.js` 中的 `SCRIPT` 数据，不依赖 `scenes.js`，但若有人尝试使用 `main.js` + `scenes.js` 的组合，会导致场景图片加载失败。
- **修复建议**：在 `ASSETS.scenes` 中添加这些 key，或将 `bg-door-*.jpg` 文件移入 `scenes/` 目录并统一命名。

## P1 — 严重影响体验

### P1-1：点击继续提示（clickContinue）永远不可见

- **严重程度**：P1
- **代码位置**：
  - `vn.html` 第 277-279 行：`.click-continue { display: none !important; }`
  - `vn.html` 第 784 行：`<div class="click-continue hidden" id="clickContinue">点击继续 ▼</div>`
  - `js/vn-engine.js` 第 4530 行：`this.elements.clickContinue.classList.remove('hidden');`
  - `js/vn-engine.js` 第 4546 行：`this.elements.clickContinue.classList.add('hidden');`
  - `js/vn-engine.js` 第 4851 行：`this.elements.clickContinue.classList.remove('hidden');`
- **问题描述**：CSS 规则 `.click-continue { display: none !important; }`（注释为"已禁用"）使得点击继续提示永远不可见。JavaScript 代码虽然通过 `classList.remove('hidden')` 尝试显示它，但 `!important` 优先级更高，无法覆盖。
- **影响**：新手玩家看到对话文本打印完毕后没有任何视觉提示，不知道需要点击对话框来推进对话。虽然点击对话框区域确实可以推进，但缺乏引导。
- **修复建议**：
  1. 删除 CSS 中的 `.click-continue { display: none !important; }` 规则（第 277-279 行）
  2. 或者使用 JavaScript 直接设置 `style.display` 来控制显示/隐藏，覆盖 CSS 的 `!important`：
     ```js
     this.elements.clickContinue.style.display = '';  // 显示时
     this.elements.clickContinue.style.display = 'none';  // 隐藏时
     ```

### P1-5：标题画面时游戏 UI 元素未被完全隐藏

## P2 — 小问题

### P2-1：`scenes.js` 和 `main.js` 是遗留代码，未在 `vn.html` 中引用

- **严重程度**：P2
- **代码位置**：
  - `js/scenes.js`（整个文件）
  - `js/main.js`（整个文件）
  - `vn.html` 第 911-930 行（script 标签列表）
- **问题描述**：`scenes.js` 和 `main.js` 使用 `SCENES` 和 `GameEngine` 的旧架构，与 `vn.html` 当前使用的 `SCRIPT` 和 `VNEngine` 架构不同。`main.js` 引用了 `sceneTitle`、`sceneMeta`、`storyText`、`protagonistReveal`、`endingBanner`、`endingSummary` 等 DOM 元素，这些元素在 `vn.html` 中不存在。如果有人在 `vn.html` 中错误地添加这些文件的引用，会导致运行时错误。
- **修复建议**：
  1. 将 `scenes.js` 和 `main.js` 移动到 `_legacy/` 目录
  2. 或者在文件顶部添加废弃注释：`// DEPRECATED: 此文件已不再使用，请使用 vn-engine.js 中的 SCRIPT 格式`

### P2-2：`scenes.js` 中的场景数据使用旧格式，与 `vn-engine.js` 中的 SCRIPT 格式不一致

- **严重程度**：P2
- **代码位置**：
  - `js/scenes.js`：使用 `bg`、`meta`、`text`、`character`、`reveal` 字段
  - `js/vn-engine.js` 第 355 行起：使用 `scene`、`dialogues`、`characters`、`onEnter` 字段
- **问题描述**：两份场景数据（`SCENES` 和 `SCRIPT`）包含相同的场景 ID（如 `start`、`meet_monk`、`monk_inquiry` 等），但使用不同的格式。`SCENES` 格式包含更丰富的文学性文本（博尔赫斯风格），而 `SCRIPT` 格式文本更简洁。这造成了维护负担——修改一个场景需要在两个地方修改。
- **修复建议**：决定一个主格式并废弃另一个。如果 `SCRIPT` 格式是当前使用的格式，应将 `SCENES` 格式中的文学性文本合并到 `SCRIPT` 中，然后废弃 `SCENES`。

### P2-3：`chapters-compat.js` 中的 `crossroads` 重定向逻辑可能造成意外行为

- **严重程度**：P2
- **代码位置**：
  - `js/chapters-compat.js` 第 55-60 行和第 166-171 行
- **问题描述**：`convertTextFormat` 和 CHAPTERS 格式转换函数中，将所有 `next: 'crossroads'` 的选择统一重定向为 `next: 'aleph_return'`。这个逻辑假设所有指向 `crossroads` 的场景都应该返回阿莱夫，但在某些上下文中，`crossroads` 可能是一个有效的场景（如序章中的 `crossroads` 场景）。如果某个旧格式章节的场景确实需要跳转到 `crossroads`，此转换会破坏游戏流程。
- **修复建议**：在重定向之前添加上下文检查，仅对 `crossroads` 有明确替换意图的场景进行重定向。或者确保所有章节场景都正确使用 `aleph_return` 而不是 `crossroads`。

### P2-4：`gameState` 初始化时 `sanity` 默认值为 100，但归档功能中可能未正确初始化

- **严重程度**：P2
- **代码位置**：
  - `js/vn-engine.js` 第 153 行：`sanity: 100`
  - `js/vn-engine.js` 第 4680 行：`gameState.sanity = Math.max(0, (gameState.sanity || 100) - choice.cost.sanity);`
- **问题描述**：`makeChoice` 中使用 `(gameState.sanity || 100)` 作为回退值，这意味着如果 `gameState.sanity` 为某个 falsy 值（如 `undefined`），会回退到 100（而非 0）。这可能导致在加载不完整存档时，理智值被错误地重置为 100 而不是 0。
- **修复建议**：使用 `??` 运算符替代 `||`：`(gameState.sanity ?? 100)`，确保只有 `null`/`undefined` 时才使用默认值。

### P2-5：`onCharacterClick` 中 `clickable.loop` 属性未使用

- **严重程度**：P2
- **代码位置**：
  - `js/vn-engine.js` 第 4817-4857 行（`onCharacterClick`）
  - `js/vn-engine.js` 第 428 行：`loop: true`（在 `meet_monk` 场景的点击角色定义中）
- **问题描述**：`SCRIPT` 数据中定义了 `clickables.monk.loop: true` 属性，表示可以反复点击角色获取随机对话。但 `onCharacterClick` 函数中并未实现循环逻辑——每次点击后 `clickInProgress` 被设为 `true`，阻止了快速连续点击，但没有实现"再次点击同一角色获取新对话"的逻辑。
- **修复建议**：在 `onCharacterClick` 中实现 `loop` 属性的支持，允许玩家在对话完成后再次点击角色触发新的随机对话。

### P2-7：`dialogueBox` 未设置 `cursor: pointer` 样式

- **严重程度**：P2
