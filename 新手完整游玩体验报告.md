### P1 — 严重影响体验

#### P1-1：点击继续提示（clickContinue）被 CSS 永久隐藏

- **代码位置**：`vn.html` 第 278-279 行
- **CSS 规则**：`.click-continue { display: none !important; }`
- **问题**：对话打字机效果结束后，玩家看不到任何"点击继续"的视觉提示。虽然点击对话框确实可以推进对话，但新手玩家不知道这个操作——他们只能靠猜。
- **JS 死代码**：`vn-engine.js` 第 4561 行 `this.elements.clickContinue.classList.remove('hidden')` 尝试显示提示，但 CSS 的 `!important` 优先级更高，此代码实际无效。

#### P1-2：类型选择按钮上的特质标签显示为英文

- **代码位置**：`vn-engine.js` 第 4640 行
- **问题**：选择按钮上显示 `methodical`（有条不紊）、`self_aware`（自我觉察）、`obsessive`（执着）等英文标签，对中文玩家完全不友好。
- **状态**：已修复（添加了 TRAIT_NAMES 映射表）

#### P1-3：标题画面时游戏 UI 未被屏幕阅读器忽略

- **代码位置**：`vn-engine.js` 第 3865-3870 行
- **问题**：标题画面通过 `opacity: 0` 隐藏游戏 UI，但元素仍在 DOM 中。使用屏幕阅读器或键盘导航的玩家可能意外触发隐藏的游戏元素。
- **状态**：已修复（添加了 `aria-hidden="true"`）

### P2 — 中等影响

#### P2-1：speakerNames 三处定义不一致

- **代码位置**：
  - `vn-engine.js` 第 4453 行（showDialogue）
  - `vn-engine.js` 第 4881 行（onCharacterClick）
  - `vn-engine.js` 第 5156 行（dialogueLog）
- **问题**：三处各有一个独立的 `speakerNames` 对象。对话日志版本（第 5156 行）包含 spike、jet、faye、ed、frodo、gandalf、aragorn、neo、trinity、tyrion、daenerys、jon、chihiro、haku、yubaba、drmanhattan、ozymandias、vladek、artie、anja 等 20 个条目，但前两处没有。
- **影响**：当前这些 speaker 尚未被任何章节使用，所以不会触发运行时错误。但这是维护隐患——如果将来任何章节使用了这些角色，对话框中会显示英文 ID 而非中文名。
- **建议**：提取为单一共享常量 `SHARED_SPEAKER_NAMES`。

#### P2-2：`scenes.js` 和 `main.js` 是遗留代码

- **代码位置**：`js/scenes.js`、`js/main.js`
- **问题**：这两个文件使用 `SCENES` 和 `GameEngine` 旧架构，`vn.html` 中未引用它们。但 `main.js` 引用了 `sceneTitle`、`sceneMeta`、`storyText`、`protagonistReveal`、`endingBanner`、`endingSummary` 等 DOM 元素，这些元素在 `vn.html` 中不存在。如果误添加引用会导致运行时错误。
- **建议**：移动到 `_legacy/` 目录或添加废弃注释。

#### P2-3：`_backup_*` 目录混乱

- **代码位置**：项目根目录下的 `_backup_v1_prologue/`、`_backup_v2_three_chapters/`、`_backup_v3_kafka_library/`、`_backup_v4_x_perspective/`
- **问题**：4 个备份目录各含 `vn.html`、`js/`、`scenes/`、`characters/`，与主项目文件极易混淆。
- **建议**：移出项目目录或加入 `.gitignore`。

#### P2-4：`gameState.sanity` 使用 `||` 而非 `??`

- **代码位置**：`vn-engine.js` 第 4680 行
- **问题**：`(gameState.sanity || 100)` 在 sanity 为 `0`（合法值）时会错误回退到 100。应使用 `(gameState.sanity ?? 100)`。
- **影响**：加载不完整存档时，理智值可能被错误重置为 100。

#### P2-5：`onCharacterClick` 未实现 `loop` 属性

- **代码位置**：`vn-engine.js` 第 4865-4920 行
- **问题**：`clickables.monk.loop: true` 定义了可重复点击角色获取随机对话，但 `onCharacterClick` 函数没有实现循环逻辑。点击一次后 `clickInProgress` 被设为 `true`，短时间内无法再次点击。
- **影响**：玩家无法如预期那样反复与角色交互。

#### P2-6：`chapters-compat.js` 中 crossroads 重定向逻辑脆弱

- **代码位置**：`js/chapters-compat.js` 第 55-60 行
- **问题**：所有 `next: 'crossroads'` 的选择被统一重定向为 `next: 'aleph_return'`。如果某个旧格式章节确实需要跳转到 `crossroads`，此转换会破坏游戏流程。
- **影响**：目前似乎没有触发此问题的场景，但未来可能成为隐患。

#### P2-7：存档系统使用 localStorage 硬编码格式

- **代码位置**：`vn-engine.js` 第 4774-4811 行
- **问题**：自动存档仅保留最近 3 个，手动存档使用 `vn_save_N` 格式。没有存档版本号，未来 `gameState` 结构变更时无法兼容旧存档。
- **建议**：添加存档版本号字段和迁移逻辑。

#### P2-8：线索通知缺少 CSS 动画定义

- **代码位置**：`vn-engine.js` 第 4833-4863 行
- **问题**：`showClueNotification` 使用 `animation: slideIn 0.5s ease` 和 `slideOut`，但 `vn.html` 的 CSS 中未定义 `@keyframes slideIn` 和 `@keyframes slideOut`。线索通知会直接出现和消失，没有过渡动画。

#### P2-9：`getDialogues()` 动态函数可能存在性能问题

- **代码位置**：多个章节文件中的 `getDialogues()`
- **问题**：部分场景使用 `getDialogues()` 函数动态生成对话（如根据 gameState 状态选择不同对话），每次调用都会重新计算。在高频点击场景下可能有轻微性能影响。
- **影响**：极小，但值得注意。

### P3 — 用户体验优化建议

#### P3-1：缺少新手引导

- 游戏没有任何教程或引导提示。新手玩家不知道：
  - 可以点击角色触发额外对话
  - 对话打字机效果可以点击跳过
  - 需要收集线索才能打开阿莱夫之门
  - 每个世界完成后可以返回阿莱夫

#### P3-2：没有音效反馈

- 点击按钮、选择选项、场景切换时没有任何音效。虽然 `music-engine.js` 和 `effects-engine.js` 存在，但音乐引擎需要在初始化时显式启动。

#### P3-3：场景切换过渡较生硬

- 场景切换只有一个 0.8 秒的黑色遮罩，没有渐变或加载动画。如果图片较大，可能出现短暂白屏。

#### P3-4：存档/读档面板确认不明确

- 保存时没有"保存成功"的确认提示。
- 读取存档时没有"是否覆盖当前进度"的二次确认。

#### P3-5：移动端适配未完成

- `vn.html` 的 CSS 未设置响应式断点，在手机上可能会出现布局错乱。
