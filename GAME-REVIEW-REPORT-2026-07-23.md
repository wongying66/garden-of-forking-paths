# 《寻找 X：小径分叉的花园》游戏评鉴报告

> 评审身份：独立游戏评鉴家 / 叙事游戏产品评审
> 评审日期：2026-07-23
> 评审版本：当前工作区版本
> 评审结论：**可玩原型，暂不建议直接作为商业成品发布**

---

## 一、先给结论

这不是一个“想法很酷但还没做出来”的项目。它已经拥有完整的浏览器运行时、12 个世界、阿莱夫枢纽、持久化游戏状态、多个结局，以及受约束的 GPT-5.6 尾声接口。技术骨架和氛围表现已经超过普通 Hackathon Demo。

但它目前更像一座**内容非常庞大的叙事资料库**，还不是一款经过充分剪辑的成熟游戏。最大问题不是缺场景，而是：

1. 玩家很难判断自己的选择究竟改变了什么；
2. 12 个世界的体验结构重复，很多路线像“换皮的哲学短篇”；
3. 文本密度和文学引用远高于交互密度；
4. “X”的谜团长期停留在抽象意象，缺少足够具体、可验证的情感关系；
5. 作品使用大量现成 IP 语汇，公开商业发布存在明显版权风险。

一句话评价：

> **它最值得保留的是“规则会记住你”的叙事框架；最需要重做的是“玩家为什么要继续做下一个选择”。**

## 二、评审依据与当前状态

### 已确认的技术规模

| 指标 | 当前结果 | 评语 |
|---|---:|---|
| 合并后的场景节点 | 836 | 规模很大，但规模本身不等于内容质量 |
| 静态场景链接 | 1,205 | 图结构完整，仍需检查分支意义 |
| 标记为结局的节点 | 162 | 其中一部分是过渡结算节点，不应直接宣传为 162 个独立结局 |
| 有效世界 | 12 | 题材覆盖广，深度不均 |
| 文本总量 | 约 209,000 字符 | 已超过一次轻量试玩应承受的体量 |
| 项目目录体积 | 约 328 MB | 角色素材约 125 MB，发布前应压缩和清理 |

### 实机检查

- 移动端 390×844 页面可正常启动。
- 序章可出现三个初始选择。
- `matrix_deny → matrix_code` 分支可继续，不会被结局页提前截断。
- 终章可进入 `final_chapter → final_return_garden → final_meet_x`。
- 结局页可显示回到阿莱夫和存档恢复入口。
- 当前浏览器加载无运行时错误；favicon 404 已清除。
- 切换场景时旧选项残留问题已修复。

这些说明**技术可运行性已经基本成立**，但不代表所有分支都完成了人工内容验收。

## 三、评分总览

| 维度 | 评分 | 判断 |
|---|---:|---|
| 核心创意 | 8/10 | “故事记住玩家”是清晰且有辨识度的卖点 |
| 氛围与视觉 | 7/10 | 色彩、背景、标题画面有统一的文学感 |
| 叙事野心 | 8/10 | 题材跨度和结构规模非常大 |
| 选择的有效性 | 4/10 | 选择常常改变文本，不足以改变玩家策略或情感判断 |
| 节奏 | 4/10 | 进入慢、文本密、世界之间重复 |
| 角色塑造 | 5/10 | 概念鲜明，但很多角色是主题的代言人而非具体的人 |
| 重玩价值 | 6/10 | 有多路线和 NG+，但路线差异还不够“值得重玩” |
| AI 功能必要性 | 5/10 | 尾声有趣，但目前更像结算后的附加服务 |
| 商业发布准备度 | 3/10 | IP、内容收束、存档和发布合规仍需处理 |

**综合判断：6/10，属于值得继续打磨的中高潜力原型。**

## 四、必须修改：发行前的 Must Have

### M1：先解决 IP 与原创性风险

当前世界直接使用或明显对应《黑客帝国》《权力的游戏》《守望者》《鼠族》《千与千寻》《指环王》《星际牛仔》《爱丽丝梦游仙境》等现成作品语汇，同时还大量使用原作角色、设定和视觉印象。

这不是“加一句致敬”就能解决的问题。若要公开商业发布，必须二选一：

- 将这些世界全部改写成原创世界，只保留主题母题；
- 明确把项目定位成非商业同人实验，并限制发布渠道和宣传方式。

建议采用第一种。比如：

- “矩阵”改成“合成避难所”；
- “中土”改成“灰烬大陆”；
- “守望者”改成“末日监察局”；
- “神隐”改成“无名浴场”；
- “太空边境”改成“失重港”。

这项不是美术润色，而是**商业发布前的硬门槛**。

### M2：让选择产生可感知、可追溯的后果

当前系统记录 traits、clues、completedWorlds 和 memories，但玩家通常看不到：

- 哪个选择改变了哪项关系；
- 哪个选择会在几场之后回来；
- 为什么这个结局是由自己的行为造成的；
- 如果当时选择另一项，未来会发生什么不同。

结果是玩家容易把选择当成“点击下一段文字”，而不是承担风险。

必须增加一个轻量的“后果回声”系统：

```text
你在第三个世界选择了隐瞒。
影响：回忆者信任 +1；监察者警戒 +1。
这项决定将在后续世界中生效。
```

不要把所有信息都公开，否则会变成数值攻略；但至少让玩家知道“这个选择被系统记住了”。

### M3：重做 12 世界的共同模板

目前大量世界都遵循：

> 进入世界 → 听一段长文本 → 选择立场 → 获得碎片 → 结局 → 返回阿莱夫。

当玩家连续完成三个世界后，这个结构会变得可预测。世界题材虽然不同，但游玩行为没有足够差异。

发行版至少需要三类不同玩法：

- **关系型世界**：通过对话、信任和隐瞒改变人物关系；
- **推理型世界**：玩家需要比较证词、记忆和矛盾信息；
- **资源型世界**：时间、理智或有限道具造成真正的取舍；
- **循环型世界**：重复进入同一世界，但系统记住上一次行为并改变内容。

每个世界不必更长，但必须在“玩家做什么”上不同。

### M4：补上新手引导和继续提示

当前 `.click-continue` 仍然被 CSS 设为 `display: none`。虽然点击对话框可以继续，但新玩家不知道该点击哪里。

首个 2 分钟必须明确告诉玩家：

1. 点击对话框可以跳过打字并继续；
2. 点击角色可能触发额外信息；
3. 选择会被记录；
4. 碎片可以解锁阿莱夫和后续世界。

引导不应做成教程弹窗。用一次带剧情的提示即可，例如：

> 雾中的文字还没说完。点击对话框，看看它愿意把什么交给你。

### M5：修复存档语义与“重新开始”

当前重新开始只清除 `vn_save_0`，没有清理 `vn_auto_save_0/1/2`。玩家完成一次旅程后点击“重新开始”，标题页仍可能保留旧的继续记录。

必须明确区分：

- 当前旅程自动存档；
- 玩家手动存档；
- NG+ 解锁状态；
- 新游戏是否保留成就。

同时给存档增加版本号和迁移逻辑，避免以后修改 `gameState` 后旧存档无法读取。

### M6：重新定义“结局数量”

当前标题动态显示“162 个结局节点”。这比旧的“92 个结局”准确，但仍然容易误导，因为其中不少是世界结算、过渡节点或带后续选择的结局标记。

建议标题只显示：

> 12 个世界 · 多重结局

后台再维护三个统计：

- terminal endings：真正终止的结局；
- world resolutions：世界结算；
- continuation endings：仍有后续分支的节点。

只有这样，玩家才不会把“节点数量”误解为独立结局数量。

## 五、目前无趣或需要大幅压缩的部分

### 1. 过度连续的哲学独白

很多文本擅长营造气氛，但经常连续解释“选择、时间、记忆、X、迷宫”的抽象含义。玩家还没有做选择，文本就已经替玩家解释了选择的意义。

这会造成两个问题：

- 玩家没有发现，而是被告知；
- 角色不像在生活，而像在朗读主题说明。

建议把每组三段独白压成一段，把剩余信息转成玩家可以调查的对象、互相矛盾的证词或可错过的细节。

### 2. “换一个文学名词”的重复感

博尔赫斯、卡夫卡、克苏鲁、矩阵、迷宫、镜子、时间循环等意象本身都很强，但密集出现时会互相削弱。玩家会逐渐感到每个世界都在用不同名字讲同一个“自我寻找”主题。

建议给每个世界一个不可替代的具体问题：

- 你是否愿意让别人记住一个不真实的你？
- 你是否会为了保护一个人而篡改证据？
- 你是否接受 X 可能根本不想被找到？

主题应该从行动中浮现，而不是由旁白反复宣布。

### 3. 只提供气质差异、不提供机制差异的选项

“勇敢 / 谨慎 / 好奇 / 沉思”这些标签能帮助系统记录角色倾向，但如果玩家只是在三种性格按钮之间选，选择会显得像人格测试。

建议减少直接标签：

```text
谨慎地询问证词
立刻拆穿他的谎言
假装相信，换取更多时间
```

让 trait 成为玩家行为的隐藏总结，而不是按钮旁边的答案提示。

### 4. 阿莱夫枢纽中的随机换门

随机展示三个世界可以制造神秘感，但也可能让玩家找不到想玩的世界，或者重复看到相似的门。随机性如果不改变剧情，只是在增加等待和点击。

建议保留“世界不一次性全部出现”的设定，但加入：

- 已看过的门记录；
- “我想找一扇与记忆有关的门”的筛选；
- 每次换门消耗或获得一个明确的信息；
- 连续两次未出现的新世界时自动保护。

### 5. 隐藏世界与全收集的奖励不够清晰

隐藏世界需要完成全部主世界并进入 NG+。对于核心玩家，这可以是奖励；对于普通玩家，它会像一个不透明的劳动合同。

必须提前告诉玩家：

- 完整收集会解锁什么；
- NG+ 会新增什么内容；
- 这不是单纯“再玩一遍看更多文本”。

## 六、需要斟酌，而不是马上删除的部分

### 1. “5 个碎片解锁终章”

这个改动改善了首轮节奏，避免玩家被迫完成 12 个世界。但它也可能让玩家忽略剩余世界。

建议保留 5 个碎片的首轮终章入口，同时把剩余世界改成：

- 终章前的可选支线；
- 终章后的不同回信素材；
- NG+ 的必要前置；
- 影响 X 回信语气和内容的证据。

不要让“5 个世界以后剩下的内容”看起来只是填充物。

### 2. GPT-5.6 回信

AI 尾声的技术边界设计是正确的：它不决定结局、不改状态、不发明世界，并且有离线备用文本。

但从玩家角度看，它现在更像“结局后生成一封漂亮的信”，还没有成为主线体验的一部分。

建议不要让 GPT 直接替代作者，而是让回信回答一个玩家确实关心的问题：

> 你最早在哪个选择里开始变成现在的自己？

同时在回信中引用一个具体选择或一个具体矛盾，而不是只复述碎片数量。

### 3. 多结局数量

多结局值得保留，但不要继续以“越多越好”为目标。玩家更在意三到六个真正不同的终局，而不是一百多个标题相近的结算节点。

建议把独立结局分成少量高质量母题：

- 找到 X，但失去自己；
- 找到自己，但接受 X 不愿回来；
- 选择成为新的引导者；
- 拒绝寻找，回到普通生活；
- 让多个世界保持分叉。

其余差异可以放在回信、片尾统计和细节场景中。

### 4. 文学引用

作品的文学气质是优点，但直接点名大量作品会让玩家感觉在参观“致敬展”。可以保留少数关键引用，其他地方改成原创意象和角色经验。

## 七、必须保留的部分

### 1. “规则记住玩家，AI只负责表达”

这是项目最清楚、最有可信度的技术叙事。规则引擎负责因果，AI 负责个性化回信，边界明确，远好于“AI 随机生成无限故事”。

### 2. 阿莱夫作为中央枢纽

阿莱夫把多个世界组织成一座心理地图，是整个项目的结构支点。它应该继续保留，但要增强“我正在构建一段旅程”的反馈，而不只是一个选门菜单。

### 3. X 的身份保持开放

不要过早解释 X 是谁。X 可以同时是一个人、记忆、未来的自己或玩家投射。但必须让这种开放性通过具体关系和事件来承载，不能只靠抽象台词维持。

### 4. 首次选择的三人结构

档案员、回忆者、追踪者是优秀的入口设计：三个角色分别代表记录、记忆和行动。建议把他们做成真正贯穿全程的镜像系统，而不只是序章的介绍人物。

### 5. NG+ 与回到花园的循环

“完成寻找后回到起点”与作品主题相符。这个循环值得保留，但 NG+ 必须有新的可见信息、新的选择或新视角，不能只是重复文本。

### 6. 离线可玩与 AI 失败降级

没有 API Key 时仍能完整游玩，并且清楚标注离线回信，这是产品可信度的重要部分，应保留。

## 八、发行优先级：MoSCoW 清单

### Must Have：不上线不能接受

1. 清理或重写现成 IP 世界。
2. 让选择的后果可见并能延迟回响。
3. 恢复“点击继续”提示，补首轮引导。
4. 修复重新开始与自动存档清理。
5. 重新定义结局统计，避免宣传虚假数量。
6. 对 12 个世界做内容删减和机制分类。
7. 给 AI 接口增加基础限流、错误监控和成本保护。

### Should Have：显著提升成品感

1. 统一所有 speaker 和 trait 的本地化字典。
2. 添加选择历史与后果回声面板。
3. 统一存档版本号和迁移机制。
4. 让阿莱夫支持筛选、已看门记录和重复保护。
5. 重写最冗长的世界，特别是文本最密集的克苏鲁和权力世界。
6. 增加声音反馈、场景过渡和保存成功提示。

### Could Have：有余力再做

1. 关系图、世界时间线和回信收藏。
2. 玩家自定义主角名字或记忆标签。
3. 多种回信风格和二周目专属信件。
4. 作者模式，让别人创建自己的世界。

### Won't Have：当前版本不要做

1. 无限生成世界。
2. 继续无上限增加结局数量。
3. 在没有原创化前继续扩充现成 IP 世界。
4. 先做复杂社交、排行榜或多人功能。

---

## 九、建议的三阶段修复路线

### 阶段 1：发行阻断项

目标：让玩家不迷路、不丢档、不被版权问题卡住。

- 首轮引导与继续提示；
- 重启 / 自动存档清理；
- 结局统计和存档版本；
- AI 接口限流；
- 确定原创化范围。

### 阶段 2：叙事剪辑

目标：减少 20%—35% 的纯解释性文本，让每个世界都留下一个具体难题。

- 每个世界保留 3—5 个关键场景；
- 删除重复的“X 是问题 / 记忆 / 镜子”解释；
- 将说明改为证据、矛盾和行动；
- 给每个世界设计独有玩法标签。

### 阶段 3：重玩和 AI 价值

目标：让第二次游玩不是重复，而是重新理解。

- 记录玩家第一次选择并在 NG+ 中回响；
- 让剩余世界影响最终回信；
- 让回信引用一个具体行为；
- 增加少量高质量终局，而不是继续堆结局节点。

---

## 十、发行前验收标准

发布前建议至少满足：

- 新玩家在 90 秒内做出第一次有意义的选择；
- 任意选择后，玩家能在 10 秒内看到“已被记录”的反馈；
- 每个世界至少有一个延迟后果；
- 每个世界的主要游玩方式至少有一项不同；
- 不出现连续三段以上、没有玩家操作的长篇说明；
- 重新开始后不会出现旧自动存档污染；
- 所有公开世界和角色名称完成原创化审查；
- AI 不可用时，玩家仍能完整完成主线；
- 每个公开宣传的“结局”都能说明它与其他结局的真实差异；
- 移动端和桌面端都完成至少一条完整路线的人工验收。

---

## 最终评语

这款游戏不应该继续沿着“再加几个世界、再加几十个结局、再加更多文学引用”的方向扩张。它现在最需要的是**删减、聚焦、让选择承担后果**。

如果只做三件事，我建议按这个顺序：

1. **把现成 IP 世界改成原创世界；**
2. **让玩家看到自己的选择如何在之后造成变化；**
3. **把 12 个世界从“同一种结构的 12 个文本包”改成“12 种不同的游玩问题”。**

做完这三件事，《寻找 X》才会从“有大量内容的文学互动 Demo”变成“有自己方法论的叙事游戏”。
